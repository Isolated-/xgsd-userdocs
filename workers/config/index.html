Config

Workers are zero config.

This is the opposite of old xGSD that is config-first.

Only a entry is required and it will not default from package.json.

Basics​

PropertyTypeDefaultNote
entrystringworker.jsrelative to cwd
diststring.xgsdrelative to cwd

Bundler​

Workers comes with a lightweight bundler to maximise compatibility.

PropertyTypeDefaultNote
enabledbooleanfalseneeds esbuild installed in your worker folder
cache.strategystringnevercan be always, change, or never

Limits​

The Worker Guard prevents a container exceeding limits.

PropertyTypeDefaultNote
ttlnumber5000the time-to-live for your container (ms)
memorynumber64maximum memory usage allowed (mb)

Output​

These options control how data is returned by a Worker.

PropertyTypeDefaultNote
modeenumwrappedcan be raw, wrapped, or auto

Example​

An example worker.json:

worker.json
{
  "entry": "worker.js",
  "dist": ".xgsd",
  "bundler": {
    "enabled": false,
    "cache": {
      "strategy": "change"
    }
  },
  "limits": {
    "ttl": 10000,
    "memory": 64
  },
  "output": {
    "mode": "wrapped"
  }
}
This experimental feature was added in @xgsd/workers@0.1.0.
Experimental features may change or be removed without notice.
