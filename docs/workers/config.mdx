**Workers** are zero config.

This is the opposite of old xGSD that is config-first.

Only a `entry` is required and it will not default from `package.json`.

---

## Basics

| Property | Type       | Default     | Note              |
| -------- | ---------- | ----------- | ----------------- |
| `entry`  | **string** | `worker.js` | relative to `cwd` |
| `dist`   | **string** | `.xgsd`     | relative to `cwd` |

---

## Bundler

**Workers** comes with a lightweight bundler to maximise compatibility.

| Property         | Type        | Default | Note                                            |
| ---------------- | ----------- | ------- | ----------------------------------------------- |
| `enabled`        | **boolean** | `false` | needs `esbuild` installed in your worker folder |
| `cache.strategy` | **string**  | `never` | can be `always`, `change`, or `never`           |

---

## Limits

The **Worker Guard** prevents a container exceeding limits.

| Property | Type       | Default | Note                                     |
| -------- | ---------- | ------- | ---------------------------------------- |
| `ttl`    | **number** | `5000`  | the time-to-live for your container (ms) |
| `memory` | **number** | `64`    | maximum memory usage allowed (mb)        |

---

## Output

These options control how data is returned by a **Worker**.

| Property | Type     | Default   | Note                               |
| -------- | -------- | --------- | ---------------------------------- |
| `mode`   | **enum** | `wrapped` | can be `raw`, `wrapped`, or `auto` |

---

## Example

An example `worker.json`:

```json title="worker.json"
{
  "entry": "worker.js",
  "dist": ".xgsd",
  "bundler": {
    "enabled": false,
    "cache": {
      "strategy": "change"
    }
  },
  "limits": {
    "ttl": 10000,
    "memory": 64
  },
  "output": {
    "mode": "wrapped"
  }
}
```
