You can use xGSD for anything from hardware tinkering to cloud-style workflows. It's a lightweight, local-first alternative to services like AWS Lambda or GCS Functions — but without the overhead of wiring everything together.

xGSD is:

- **Local-first** — runs inside your Node.js project
- **Function-driven** — every unit is a function with a consistent shape
- **Predictable** — built-in execution rules (retry, timeout) remove guesswork
- **Extensible** — add behaviour through plugins, not config hacks
- **Low learning curve** — no frameworks, just Node.js with `xgsd run`

<Concept>
At its core, xGSD runs functions inside your Node.js project.

Each function:

- accepts a single argument
- returns an object, `null` or `undefined`

```typescript
type RunFn<T, R> = (data: T): Promise<R>
```

After creating a function, you can use `xgsd call` or `xgsd run` to call it.

</Concept>

### 1. Install

<ApiTabs npm={`npm install -g @xgsd/cli`} yarn={`yarn global add @xgsd/cli`}>
  Install xGSD globally:
</ApiTabs>

To verify the installation:

```bash
xgsd version
```

You should see something like:

```bash
@xgsd/cli/0.5.0 linux-x64 node-v24.4.1
```

---

### 2. Create a function

:::info

You can use **ESM**, **CommonJS**, or **TypeScript** for your functions.

If you're using **TypeScript**, ensure you import files with the full extension (e.g. `file.ts` not `file`).

:::

<ApiTabs
  js={`export async function greet({ name }) {
    return { message: \`Hello \$\{name\}\` }
}`}
  ts={`export async function greet({name}: { name: string }) {
    return { message: \`Hello \$\{name\}\` }
  }`}
  filename="index"
>
  Export a function:
</ApiTabs>

xGSD loads functions from your project's entry point (usually defined in `package.json` or `config.yaml`).

---

### 3. Call it

Run your function with:

```bash
xgsd call greet -d '{"name": "World"}'
```

Output:

```bash
{
  "message": "Hello World"
}
```
