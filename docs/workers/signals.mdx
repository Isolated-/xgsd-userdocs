**Signals** are streamed from the container your worker runs in.

They're a collection of:

- **Activations**
- **Errors** and **warnings**
- **Logs**
- **Metrics**
- **System messages**

By default signals are appended to `process.stdout` but you can override it by setting `stream` when calling `createTransport`.

:::tip
When developing, `stream: process.stdout` can be very useful as it'll print information to your console vs file.

It also provides:

```bash
node run.js | grep "system"
```

:::

---

## Format

The format is consistent:

```json
{
  "ctx": "{context_id}",
  "act": "{activation_id}",
  "pid": "{process_id}",
  "type": "log|error|activation|system|generic",
  "message": "{message}",
  "meta": {},
  "timestamp": 1778318464737
}
```

for all **Signals** and `meta` reflects any signal-specific data.

:::info
`act` and `pid` were added in **V1.1**.
:::

---

## Sending your own Signals

You can send **Signals** to `stdout`/`stderr`.

Messages will be picked up and marked with `generic` unless you use the format above.

You can also define your own format using `generic` signals, for example:

```js
console.log(
  JSON.stringify({
    production: false,
    name: "my worker",
    description: "just started",
  }),
);
```

This will result in a message like:

```json
{
  "ctx": "{id}",
  "type": "generic",
  "message": "generic",
  "meta": {
    "production": false,
    "name": "my worker",
    "description": "just started"
  },
  "timestamp": 123
}
```

So you can build on **Signals** to better match your app/environment.

:::tip
**Signals** are in JSON format.

If you use `stream: process.stdout` in development it can be hard to tell what's going on.

You can clean this up by doing something like this:

```js
const transport = createTransport({
  entry: "./worker.js",
  stream: {
    write: (chunk) => {
      const json = JSON.parse(chunk);
      console.log(json.message);
    },
  },
});
```

That way you go from:

```text
{"ctx":"504fabe1","type":"system","message":"starting container",...}
```

To:

```text
starting container
```

:::
