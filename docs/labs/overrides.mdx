## Overriding default behaviour

You can override default behaviour by creating a `xgsd.yaml` file in `~/.config/xgsd`.

You can also create a `xgsd.yaml` inside your project to apply per-project overrides.

```yaml
# configure code bundling (replaces CLI flags)
bundler:
  enabled: true
  cache:
    strategy: never # auto | change | never

# configure runtime (replaces CLI flags)
runtime:
  save: false
  debug: false
  process:
    enabled: false

# metrics can be globally accepted/rejected
# vs defining it in your config.yaml
metrics:
  enabled: false
```

Minimal support was added in **v0.6.4** - this will be extended.
