Creating a Project

In Getting Started, you created and called a single function with xgsd call.

Now you'll turn that into a full project using chain mode.

Concept (TLDR)

A Project defines how functions are executed in sequence through configuration.

It controls how data flows between them and how execution is managed.

Chain mode ensures the next block receives the output of the last.

1. Create a project folder​

Create a project folder:

mkdir my-project
cd my-project
  • yaml
  • JSON

Create a config file and copy and paste the following:

config.yaml
mode: chain
data:
  name: World
blocks:
  - run: greet
  - run: save

Create a config file and copy and paste the following:

config.json
{
  "mode": "chain",
  "data": {
    "name": "World"
  },
  "blocks": [
    {
      "run": "greet"
    },
    {
      "run": "save"
    }
  ]
}

2. Add a new function​

Create an index.js file inside your project folder.

  • Node.js

Make sure greet is exported:

index.js
export async function greet({name}) {
  return { message: `Hello ${name}`}
}
  • Node.js

Now create a new exported function save:

index.js
export async function save({message}) {
  writeFileSync("message.txt", message);
}

In chain mode, each function receives the output of the previous one.

So save automatically receives the message from greet.

3. Run it​

Run the project:

xgsd run

Optionally override the config data:

xgsd run -d '{"name":"Me"}'

4. View the result​

xGSD stores run results in your project's runs folder.

After execution you'll see:

  • greet produced a message
  • save wrote it to message.txt

Open message.txt:

message.txt
Hello World

That's your first chained result.

5. Templating alternative​

You don't need a function for every step.

Instead of using greet, you can use Templating directly:

  • yaml
  • JSON
config.yaml
mode: chain
data:
  name: World
blocks:
  - run: save
    input: 
      message: Hello {{ctx.data.name}}
config.json
{
  "mode": "chain",
  "data": {
    "name": "World"
  },
  "blocks": [
    {
      "run": "save",
      "input": {
        "message": "Hello {{ctx.data.name}}"
      }
    }
  ]
}

This produces the same result — without introducing an extra function.
