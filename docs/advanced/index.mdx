## Setup

The guides in this section will reference a `setup` function.

This is a function you export in your code. You use it to register plugins, executors, and orchestrators:

```javascript title="index.js"
export function setup(xgsd) {
  xgsd.use(MyPlugin);
  xgsd.executor(MyExecutor);
  xgsd.orchestrator(MyOrchestrator);

  // you can also provide instances:
  xgsd.use(new MyPlugin(opts));

  // or factory:
  xgsd.use((ctx) => new MyPlugin(ctx));
}
```

## Bootstrap

When building your own apps to run projects (replacing the CLI/API) you may want to use `bootstrap()` directly.

Here's how to do it (simplified):

```javascript title="index.js"
// imports
import {
  bootstrap,
  createConfig,
  DefaultOrchestrator,
  ProcessExecutor,
} from "@xgsd/runtime";

// inside some async function
const config = createConfig({
  packageJsonPath: "/path/to/project/package.json",
  configPath: "/path/to/project/config.yaml",
  validator: (input) => input,
});

// run the project with bootstrap()
const result = await bootstrap({
  projectPath: "/path/to/project",
  config,
  activation: "custom",
  preset: {
    plugins: [],
    executor: ProcessExecutor,
    orchestrator: DefaultOrchestrator,
  },
});
```
