### Concept

**Environment variables** are a standard way to configure behaviour in xGSD.

They can be provided either directly or resolved from external sources.

---

### 1. Static variables

Static values can be defined directly in your `config.yaml`:

```yaml
# inside your blocks:
env:
  MY_VAR: 1234
```

This is suitable for non-sensitive configuration.

---

### 2. Resolved variables

Sensitive values can be resolved at runtime instead of being stored in your config.

```yaml
# inside your blocks:
env:
  MY_VAR:
    source: env
    ref: SENSITIVE
```

This reads from `process.env.SENSITIVE`.

You can also load them from files:

```yaml
# ... block config
env:
  MY_VAR:
    source: file
    ref: my_var
    path: myvars.json
```

```yaml
# ... block config
env:
  MY_VAR:
    source: file
    ref: my_var
    path: myvars.env
```

Resolved values are injected into the runtime environment.

### 3. xGSD variables

:::warning
**Running with `--local` or `--dev`?** Use `export SENSITIVE=1234567` and read it with `process.env.SENSITIVE` instead.
:::
