> This is an **experimental feature** added in **v1.0** designed to make running projects automatically easier.
>
> You can manage the server with `xgsd up|down|status`.
>
> Start the server and send a POST request to `http://localhost:3010/run` with `{"project": "/abs/path/to/project"}` to run a project.
>
> It is not designed to be exposed publicly.

---

### Authentication

When you run `xgsd up`, you'll generate a new token:

```text
Run: http://localhost:3010/run
Info: http://localhost:3010/info
Unique token: 80ee0b98-b544-4ab0-be73-ca8031daa867 (Bearer)
```

To use this token in a request:

```bash
curl -X POST http://localhost:3010/run \
  -H "Authorization: Bearer 80ee0b98-b544-4ab0-be73-ca8031daa867" \
  -H "Content-Type: application/json" \
  -d '{"project":"/abs/path/to/project"}'
```

If you don't want to generate a new token every time you run `xgsd up`, provide one with `--token`.

You can skip authentication entirely by using `--no-auth` making development easier.
